Week 4 (25/02 - 01/03): ontwerpen en beginnen met realiseren

De week

Maandag ochtend ben ik begonnen met het inzien van de Azure omgeving voor het opzetten van een database omgeving. Echter heb ik ervoor gekozen om voor mijn ontwikkel omgeving een database lokaal te hosten en een gratis versie te hosten op MongoDB atlas. Het probleem wat hierbij kwam kijken is dat in Veenendaal alleen port 443 wordt geaccepteerd en de database servers altijd op 21017 draaien waardoor ik hier in Veenendaal niet mee kan verbinden. Mocht het nodig zijn om dit op te lossen dan kan ik dat via mijn opdrachtgever doen. Verder ben ik deze dag bezig geweest met de basis op te zetten van mijn react native app en heb ik best practices opgezocht. Als laatste heb ik die dag een bedrijfsbezoek gehad van mijn begeleider van School samen met Hans Geurtsen en Bart Bijl. Tijdens dit gesprek heb ik mijn officiele GO gekregen!

Dinsdag ben ik begonnen met het ontwikkelen van de eerste functionaliteit van de app. Op de backend (API) heb ik een call ontwikkeld waarbij een nieuwe gebruiker zich kan aanmelden. Na deze ontwikkeling ben ik van start gegaan met een react native course om in een paar dage tijd met een goede basis te beginnen.

Vandaag verder gegaan met waar ik gisteren (dinsdag) was gebleven met de react native course. Hier heb ik wat problemen ondervonden met het installeren van één van packages. De versie van React Native die ik gebruikte werd door deze package niet ondersteund maar dit stond nergens gedocumenteerd. Eenmaal uitgevonden kon ik verder. Na de pauze had ik een gesprek met Marco Kuipers wie erg veel kennis heeft over mobile development. De notities van dat gesprek kun je hier vinden: Notities gesprek. Aan het eind van de werkdag heb ik weer met een groot deel van de collega's gegeten en een ISKA (Info Support Kennis Avond) gevolgd. Deze sessie ging over het ontwerpen van code waarbij je makkelijk frameworks uit kunt wisselen. Een groot deel van deze ISKA ging over Clean Architecture, een architectuur die ik ook gebruik voor het ontwikkelen van mijn backend.

Donderdag heb ik de hele dag gebruikt voor de react native course.

In de ochtend heb ik op vrijdag de react native course afgemaakt waarna ik daarna kon beginnen aan het ontwikkelen van de eerste functie. Op dinsdag had ik de API call voor het aanmaken van een nieuwe gebruiker al geïmplementeerd en vandaag was ik begonnen met het ontwikkelen van de front-end hiervan. De UI hiervan is bijna af waarna op maandag de daadwerkelijke API eraan gekoppeld zal worden.

Problemen en feedback

Ik ben tijdens deze week tegen twee problemen aangelopen. Ten eerste kon ik geen verbinding maken met een database die online gehost staat omdat ze in Veenendaal alle porten dicht houden. Echter kan dit probleem opgelost worden door met een lokale database te werken en server gerelateerde dingen te testen in Den Bosch of thuis. Daarnaast werkte een react native package niet lekker mee doordat de react native versie waar ik mee werk niet ondersteund werd. Hier heb ik van geleerd en zal voortaan één van de eerste dingen zijn waar ik naar zal kijken om dat gelijk uit te sluiten.
